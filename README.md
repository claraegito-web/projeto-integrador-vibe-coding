Fase 1 do Projeto Integrador II

## 🎯 1. Definição do Problema
O desenvolvimento de software moderno exige rapidez, mas muitos estudantes e devs juniores travam na hora de quebrar um grande problema em pequenas tarefas técnicas. O **VibeTasks** utiliza o conceito de **Vibe Coding** para transformar uma intenção de alto nível em um backlog de tarefas acionáveis automaticamente.

## 👤 2. Personas (IA Driven)
* **Enzo, 19 anos (Estudante de ADS):** Sente-se paralisado ao iniciar um novo projeto. Ele sabe o que quer criar (ex: um app de clima), mas não sabe a ordem lógica das pastas e arquivos.
* **Mariana, 30 anos (Transição de Carreira):** Estuda à noite e tem pouco tempo. Ela precisa de uma IA que resuma o que falta fazer no código para ela não perder o "fio da meada" entre um dia e outro.

## 💡 3. Proposta de Valor & Vibe Coding
O foco do VibeTasks não é apenas gerenciar cards, mas **gerar contexto**.
* **Intenção sobre Sintaxe:** O usuário descreve a "vibe" do que quer codar.
* **Tech Lead Assistente:** A IA sugere as tecnologias e os passos iniciais para o MVP.
* **Fluidez:** Interface minimalista para evitar distrações.

## 🚀 4. Backlog de Funcionalidades (MVP)
1.  **Input de Intenção:** Campo de texto livre para descrever o projeto.
2.  **IA Task Generator:** Quebra do texto em 5 sub-tarefas técnicas.
3.  **Kanban Minimalista:** Colunas de 'Para fazer', 'Fazendo' e 'Pronto'.
4.  **Resumo de Progresso:** Relatório diário gerado por IA para o professor ou mentor.

## 📁 5. Organização do Repositório
* `/docs`: Documentação de ideação e registro de prompts.
* `/frontend`: Estrutura da interface (Vite/React).
